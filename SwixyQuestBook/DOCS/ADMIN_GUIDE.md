# Questbook — Админ-панель: руководство

## Открытие админки

1. Откройте книгу квестов клавишей **K**.
2. Внизу справа появится кнопка **настроек** (шестерёнка). Нажмите её.
3. Откроется админ-панель в левой части окна.

> Требуется привилегия `controlserver` на сервере.

---

## Интерфейс админки

```
┌─────────────────────────┐
│  Добавить элемент        │  ← заголовок
├─────────────────────────┤
│  [Пресет ▼]             │  ← выбор типа узла
│  START | QUEST | CHECKPOINT │
├─────────────────────────┤
│  [1] [2] [3] [4] [5]   │  ← слоты (только для QUEST)
├─────────────────────────┤
│  Goals_ID_1: [________] │
│  Goals_Num_1: [___]     │
│  Goals_ID_2: [________] │
│  Goals_Num_2: [___]     │
│  Goals_ID_3: [________] │
│  Goals_Num_3: [___]     │
│  Goals_ID_4: [________] │
│  Goals_Num_4: [___]     │
│  Awards_ID_1: [________]│
│  Awards_Num_1: [___]    │
│  Awards_ID_2: [________]│
│  Awards_Num_2: [___]    │
├─────────────────────────┤
│  Information text:       │
│  [____________________] │
├─────────────────────────┤
│  Direction: R            │  ← для QUEST/CHECKPOINT
│  Preset quests: 1        │  ← количество детей (1/2/3/5)
│  Parent Node: AUTO       │  ← привязка к узлу
├─────────────────────────┤
│  [Add] [Delete]          │
│  [Clear]                 │
│  [Save]                  │
└─────────────────────────┘
```

## Пресеты узлов

### START
- Начальный узел категории.
- Всегда позиционируется в (0,0).
- Только **Information text** (до 624 символов).
- **Нет** Direction, Parent Node, Preset quests.

### QUEST
- Обычный квест.
- До **4 предметов-целей** (Goals_ID + Goals_Num).
- До **2 наград** (Awards_ID + Awards_Num).
- **Information text** (до 165 символов).
- **Direction** (T/R/B/L).
- **Preset quests** (1/2/3/5) — сколько дочерних квестов будет создано.
- **Parent Node** — к какому узлу привязать.
- **5 слотов** — для создания нескольких квестов за раз.

### CHECKPOINT
- Контрольная точка без предметов.
- **Information text**.
- **Direction**.
- **Parent Node** — привязка к узлу.

---

## Поля ввода

| Поле | Тип | Описание |
|------|-----|----------|
| `Goals_ID_1-4` | Текст | Код предмета (например `stick`, `game:nugget-nativecopper`). Авто-префикс `game:`. |
| `Goals_Num_1-4` | Число | Количество предмета (1-64). |
| `Awards_ID_1-2` | Текст | Код награды. |
| `Awards_Num_1-2` | Число | Количество награды (1-64). |
| `Information text` | Текст | Описание квеста (Quest: 165, Start: 624 символов). |
| `Direction` | Переключатель | T (Top), R (Right), B (Bottom), L (Left). Клик переключает. |
| `Preset quests` | Переключатель | 1, 2, 3 или 5 детей. Клик переключает. |
| `Parent Node` | Переключатель | AUTO или конкретный ID узла. Клик переключает. |

### Горячие клавиши в полях

- **Tab** — переход к следующему полю.
- **Backspace** — удаление символа.
- **Ctrl+V** — вставка из буфера.
- **Escape** — снятие фокуса с поля.

---

## Слоты (кнопки 1-5)

Только для пресета **QUEST**. Позволяет подготовить до 5 квестов одновременно.

- Каждый слот хранит свои Goals/Awards/InfoText/Direction.
- При нажатии **Add** — все заполненные слоты добавляются как отдельные узлы.
- Слоты автоматически сохраняются при переключении.

---

## Направления (Direction)

Направление определяет, куда будет размещён новый узел относительно родительского:

| Direction | Описание | Угол |
|-----------|----------|------|
| **T** (Top) | Вверх | 0° |
| **R** (Right) | Вправо | 0° (вправо) |
| **B** (Bottom) | Вниз | 180° |
| **L** (Left) | Влево | 180° (влево) |

> Side-ветки (ветвление) идут по диагонали 45° от основного направления.

---

## Preset quests — количество детей

| Preset | Кол-во детей | Схема размещения |
|--------|-------------|------------------|
| **1** | 1 квест | Forward (прямо) |
| **2** | 2 квеста | Side Right + Side Left (по диагонали) |
| **3** | 3 квеста | Forward + Side Right + Side Left |
| **5** | 5 квестов | Forward + Side Right + Side Left + Sub-Left + Sub-Right |

---

## Кнопки действий

| Кнопка | Действие |
|--------|----------|
| **Add** | Добавляет узлы из заполненных слотов. |
| **Delete** | Удаляет последний добавленный узел. |
| **Clear** | Очищает все поля и сбрасывает состояние. |
| **Save** | Сохраняет изменения на сервере и рассылает всем игрокам. |

## Примеры создания веток квестов

### Пример 1: Простая линейная ветка (3 квеста подряд)

**Цель:** Start → Квест 1 → Квест 2 → Квест 3

**Шаг 1 — Создаём Start:**
1. Пресет: **START**
2. Information text: `Добро пожаловать в мир квестов!`

**Шаг 2 — Создаём Квест 1:**

2. Goals_ID_1: `stick`, Goals_Num_1: `5`

4. Information text: `Собери 5 палок`
5. Direction: **R**
6. Preset quests: **1**
7. Parent Node: **AUTO** (привяжется к последнему узлу)
8. Нажмите **Add**

**Шаг 3 — Создаём Квест 2:**
1. Пресет: **QUEST**
2. Goals_ID_1: `game:nugget-nativecopper`, Goals_Num_1: `3`
3. Awards_ID_1: `game:iron-ingot`, Awards_Num_1: `1`
4. Information text: `Добуди 3 самородка меди`
5. Direction: **R**
6. Preset quests: **1**
7. Parent Node: **AUTO**
8. Нажмите **Add**

**Шаг 4 — Создаём Квест 3:**
1. Пресет: **QUEST**
2. Goals_ID_1: `game:iron-ingot`, Goals_Num_1: `2`
3. Awards_ID_1: `game:steel-ingot`, Awards_Num_1: `1`

### Пример 3: Максимальное ветвление (Preset 5)

**Цель:** Один квест ветвится на 5 направлений.

**Шаг 1 — Создаём Start:**
1. Пресет: **START**
2. Information text: `Великое приключение начинается!`
3. Нажмите **Add**

**Шаг 2 — Создаём квест с Preset 5:**
1. Пресет: **QUEST**
2. Goals_ID_1: `stick`, Goals_Num_1: `1`
3. Information text: `Выбери свою судьбу`
4. Direction: **R**
5. Preset quests: **5**
6. Parent Node: **AUTO**
7. Нажмите **Add**

Схема размещения (от квеста):
```
        [Sub-Left]
            ↖
[Side-Left]   [Forward]   [Side-Right]
            ↙
        [Sub-Right]
```

5 направлений:
- Forward (прямо) — 0°
- Side Right (+45°) — диагональ вправо-вверх
- Side Left (-45°) — диагональ влево-вверх
- Sub-Left (+90°) — влево-вверх (перпендикулярно)
- Sub-Right (-90°) — вправо-вверх (перпендикулярно)

**Шаг 3 — Сохраняем:**
Нажмите **Save**.

## Wildcard-предметы (шаблоны)

Код предмета может содержать `*` для поиска по шаблону:

| Шаблон | Описание | Пример совпадения |
|--------|----------|-------------------|
| `clay-*` | Начинается с `clay-` | `clay-blue`, `clay-red` |
| `*-stick` | Заканчивается на `-stick` | `bone-stick`, `iron-stick` |
| `*clay*` | Содержит `clay` | `clay-blue`, `fireclay` |
| `stick` | Точное совпадение | `stick` |

Проверка без учёта регистра (OrdinalIgnoreCase).

---

## Сохранение и загрузка

1. **Save** — отправляет все изменения на сервер.
2. Сервер сохраняет в `quests.json` и рассылает обновление всем игрокам.
3. **ESC** или повторное нажатие **K** — отменяет несохранённые изменения (восстанавливает снэпшот до открытия админки).

---

## Советы

- Используйте **слоты** для подготовки нескольких квестов за раз — это быстрее, чем добавлять по одному.
- **Parent Node: AUTO** привяжет к последнему добавленному узлу. Используйте конкретный ID, если нужно привязать к определённому узлу.
- **Direction** влияет только на позицию нового узла на графе — не на логику разблокировки.
- Нажмите **ESC** в поле ввода, чтобы снять фокус и вернуться к навигации.
- После каждого важного изменения нажимайте **Save**, чтобы не потерять работу.
